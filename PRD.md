# 塔罗牌在线抽牌 — 产品需求文档 (PRD)

> 版本：v1.0 | 日期：2026-07-16 | 状态：待评审
> 技术架构请参阅：[architecture.md](./architecture.md)

---

## 1. 产品概述

### 1.1 产品定位

一款**极简的在线塔罗牌抽牌工具**。用户无需了解塔罗牌知识，只需选择牌阵、输入问题、点击抽牌并翻开卡牌，即可获得牌面解释和针对问题的简要分析。最终支持 PWA 安装到手机桌面，随时随地使用。

### 1.2 目标用户

- 对塔罗牌好奇但零基础的普通用户
- 有具体问题想要寻求方向性建议的人
- 需要随时随地快速获得塔罗体验的用户（手机端优先）

### 1.3 核心痛点

- 传统塔罗占卜需要找塔罗师，有时间和金钱成本
- 实体塔罗牌不便携，物理抽牌缺少私密性
- 现有塔罗 App 大多功能臃肿、注册繁琐

### 1.4 产品目标

- 3 步完成一次占卜：选牌阵 → 输问题 → 翻牌看结果
- 无需注册、无需付费、打开即用
- 移动端优先，最终支持 PWA 安装到手机桌面

## 2. 功能需求

### 2.1 牌阵选择

提供 3 个经典牌阵，覆盖不同的问题类型：

#### 牌阵一：三张牌（时间之箭）

| 位置 | 含义 |
|------|------|
| 第 1 张（左） | 过去 — 已发生的事件或根源 |
| 第 2 张（中） | 现在 — 当前状况或核心挑战 |
| 第 3 张（右） | 未来 — 可能的发展方向 |

- **适用场景**：通用型，适合大多数问题
- **牌数**：3 张

#### 牌阵二：六芒星（星芒指引）

七张牌以六芒星形状排列（中心一张，六个顶点各一张）：

| 位置 | 含义 |
|------|------|
| 第 1 张（中心） | 核心 — 问题的本质或当前核心状态 |
| 第 2 张（顶部） | 目标 — 期望的结果或理想方向 |
| 第 3 张（右上） | 助力 — 可利用的资源或支持力量 |
| 第 4 张（右下） | 阻碍 — 面临的障碍或需克服的困难 |
| 第 5 张（底部） | 根源 — 问题的深层原因或潜意识影响 |
| 第 6 张（左上） | 建议 — 塔罗给出的行动指引 |
| 第 7 张（左下） | 结果 — 按当前路径发展的最终走向 |

- **适用场景**：复杂问题的深度剖析
- **牌数**：7 张

#### 牌阵三：二选一（抉择之路）

五张牌分两路排列，帮助用户比较两个选项：

| 位置 | 含义 |
|------|------|
| 第 1 张（中心） | 现状 — 当前面临选择的核心状态 |
| 第 2 张（左侧上） | 选项 A 的优势 — 选择路径 A 的积极面 |
| 第 3 张（左侧下） | 选项 A 的挑战 — 选择路径 A 的代价/风险 |
| 第 4 张（右侧上） | 选项 B 的优势 — 选择路径 B 的积极面 |
| 第 5 张（右侧下） | 选项 B 的挑战 — 选择路径 B 的代价/风险 |

- **适用场景**：在两个选项之间犹豫不决时
- **牌数**：5 张
- **特殊交互**：抽牌前需分别输入选项 A 和选项 B 的名称

### 2.2 问题输入

- 提供文本输入框，placeholder 示例：「我的感情发展会如何？」「这份工作该不该接受？」
- 问题**可选填**（不填则仅看牌面含义，不做针对性分析）
- 二选一牌阵需额外输入两个选项名称（必填）

### 2.3 抽牌交互

1. 用户点击「开始抽牌」按钮
2. 系统从 78 张牌中**随机**抽取对应张数（每张牌带正/逆位随机状态）
3. 已抽出的牌以**牌背朝上**的方式排列在牌阵对应位置
4. 抽牌过程伴有简单的洗牌动画（牌背快速轮转 1-2 秒）
5. 不支持重复抽牌（需点击「重新抽牌」回到牌阵选择页）

### 2.4 翻牌交互

- 用户**逐张点击**卡牌翻开
- 翻牌动画：CSS 3D `rotateY(180deg)` 翻转，模拟真实翻牌手感
- 翻开后显示：
  - 牌名（如「愚者」「圣杯三」）
  - 正位 / 逆位标识
  - 牌面符号（CSS/SVG 绘制）
  - 牌面含义简述（≤ 30 字）
- 已翻开的牌可随时回看，不影响进度
- 所有牌翻完后，自动显示综合分析区域

### 2.5 结果展示

#### 单张牌解释
每张牌翻面后展示，字数控制在 **30 字以内**：
- 牌名
- 正/逆位标签
- 一句话含义（如「新的开始，带着勇气踏上未知的旅程」）

#### 综合分析
所有牌翻完后汇总展示：
- 基于牌阵位置含义 + 各牌正逆位 + 用户问题的**模板拼接分析**（约 150-300 字）
- 分析结构：
  1. 问题回顾
  2. 牌面整体能量概述（正位多/逆位多/大阿卡纳多等）
  3. 关键位置解读 + 牌面之间的关联
  4. 简要建议收尾

#### AI 深度分析引导
在综合分析下方展示提示：
> 💡 想要更个性化的深度解读？复制下方信息，粘贴到豆包（doubao.com）、DeepSeek 等 AI 助手中提问，获得更详细的分析。

并提供「一键复制」按钮，复制内容格式为：

```
我正在使用塔罗牌占卜，我的问题是：「{用户问题}」。
使用的牌阵是「{牌阵名称}」，抽到的牌如下：
- 位置1（{位置含义}）：{牌名}（{正/逆位}）
- 位置2（{位置含义}）：{牌名}（{正/逆位}）
...
请帮我解读这些塔罗牌的含义，并结合我的问题给出综合分析。
```

### 2.6 PWA 能力（最终目标）

- Service Worker 缓存全部静态资源，**离线可用**
- Web App Manifest 配置，支持**添加到主屏幕**
- 图标适配 iOS/Android（192px + 512px）

## 4. UI/UX 设计

### 4.1 视觉风格

**方向**：华丽金黑风格，灵感来自奢华塔罗牌桌与古典占卜室

- **主色调**：深黑底 `#0d0d0d`，璀璨金 `#c9a050` / `#e0c878`
- **辅助色**：暗酒红 `#5c1a2a`，深紫 `#2a1a3c`，奶油白 `#f5eedc`
- **背景**：羊毛毯纹理（CSS `repeating-conic-gradient` 模拟编织纹理），叠加暗角 vignette 效果
- **字体**：标题使用衬线字体（如 Cormorant Garamond / Playfair Display），正文使用无衬线系统字体
- **氛围元素**：
  - 页面边缘金色边框装饰线
  - 角落星光粒子缓慢飘浮
  - 卡牌周围柔光光晕（`box-shadow` 金色 glow）
  - 蜡烛火焰动画（可选装饰）
- **卡牌背面**：统一图案 — 深红底色 + 金色几何星芒纹样 + 边框描金，CSS 绘制
- **牌面**：羊皮纸色底 + 黑金花纹边框 + 花色符号（SVG 绘制）+ 数字/牌名

### 4.2 页面流程

```
┌─────────┐    ┌──────────┐    ┌──────────┐    ┌────────┐    ┌──────────┐
│  首页    │ →  │ 选择牌阵  │ →  │ 输入问题  │ →  │  抽牌   │ →  │ 翻牌结果  │
│ 欢迎页   │    │ 3个选项   │    │ (可选)    │    │ 动画    │    │ 逐张翻开  │
└─────────┘    └──────────┘    └──────────┘    └────────┘    └──────────┘
                                                                    │
                                                                    ▼
                                                              ┌──────────┐
                                                              │ 综合解读  │
                                                              │ +AI提示   │
                                                              └──────────┘
```

### 4.3 页面详细设计

#### 页面 1：首页
- 大标题「塔罗指引」，金色衬线字体
- 副标题：「用心提问，牌面会给你答案」，奶油白小字
- 背景羊毛毯纹理 + 暗角
- 装饰：顶部星月 SVG，底部金色分割线
- 大按钮：「开始占卜」→ 金色描边，hover 发光
- 底部小字：「无需注册，完全免费」

#### 页面 2：牌阵选择
- 3 张选择卡片，金色边框，悬停发光
- 卡片内容：牌阵名称、牌数、适用场景、布局简图
- 选中态：金色内发光
- 底部「返回首页」文字链接

#### 页面 3：问题输入
- 居中大输入框，深色底 + 金色边框
- 柔和 placeholder 文字
- 二选一牌阵时额外显示「选项 A」「选项 B」输入框
- 「开始抽牌」按钮，金色渐变
- 底部「返回修改牌阵」链接

#### 页面 4：抽牌动画
- 牌背快速交替闪现（约 1.5 秒）
- 动画结束后，牌以牌阵布局排列（牌背朝上）
- 每张牌下方标注位置标签（金色小字）
- 引导文字：「点击卡牌翻开」

#### 页面 5：翻牌 & 结果
- 点击某张牌 → 3D 翻转动画 → 展示牌面内容
- 牌面：羊皮纸质感底色 + 符号图案 + 牌名 + 正逆位 + 一句话含义（≤30字）
- 已翻牌保持亮态，未翻牌牌背 + 金色呼吸光效提示点击
- 全部翻完后，底部滑入综合解读卡片
- 解读卡片下方：AI 引导提示 + 「一键复制」按钮
- 底部：「重新占卜」按钮

### 4.4 动画设计

| 动画 | 实现方式 | 时长 |
|------|----------|------|
| 洗牌 | 牌背快速轮播 + scale 变化 | ~1.5s |
| 翻牌 | CSS `transform: rotateY(180deg)` + `backface-visibility` | ~0.5s/张 |
| 牌面出现 | fadeIn + translateY | ~0.3s |
| 综合解读入场 | slideUp + fadeIn | ~0.4s |
| 金色粒子 | CSS animation 循环飘浮 | 持续 |
| 未翻牌呼吸提示 | `box-shadow` 金色脉冲动画 | 循环 2s |

### 4.5 响应式设计

- **移动端（<640px）**：牌阵竖排、卡片全宽、牌阵布局缩放适配
- **平板（640-1024px）**：牌阵可横排、卡片适当放大
- **桌面（>1024px）**：居中容器 max-width 800px，两侧留白展示羊毛毯纹理

## 5. 项目规划

### 5.1 开发阶段

| 阶段 | 内容 | 预计产出 |
|------|------|----------|
| **Phase 1** | 项目初始化 + 数据准备 | Vite 项目搭建、78 张牌数据录入、类型定义 |
| **Phase 2** | 核心交互 | 牌阵选择、问题输入、洗牌抽牌、翻牌动画 |
| **Phase 3** | 解读系统 | 单牌解释展示、分析模板拼接引擎、AI 引导 |
| **Phase 4** | 视觉打磨 | 金黑塔罗风格 CSS、卡牌图案 SVG、背景纹理、粒子 |
| **Phase 5** | PWA + 响应式 | Service Worker、Manifest、移动端适配 |
| **Phase 6** | 测试 + 部署 | 功能验证、性能优化、部署到静态托管 |

### 5.2 里程碑

- **M1** — Phase 1+2 完成：可完整走通选牌阵→抽牌→翻牌流程
- **M2** — Phase 3 完成：可看到单牌解释和综合分析
- **M3** — Phase 4+5 完成：视觉完整、可 PWA 安装
- **M4** — Phase 6 完成：上线发布

### 5.3 后续迭代方向（v2+）

- 更多牌阵（如凯尔特十字、单张牌）
- 每日塔罗运势卡片
- 保存抽牌历史（localStorage）
- 语音输入问题
- 多语言支持（英文版）
